## 蒸馏规则

### 素材类型与处理方式

| 素材来源 | 处理方式 | 说明 |
|---------|---------|------|
| **论坛帖子** | 蒸馏转化为 Markdown | 因帖子格式混乱需重排，但**必须保证内容完整** |
| **文件/文件夹**（skill、参考文档等） | 原样保存，建文件夹放入 | 格式没问题就**不做改动**，直接保留原始结构上传 |

### 核心原则

1. **完整性 > 格式美化** — 宁可格式粗糙也不能丢内容
2. **非论坛文件默认原样保存** — 可能是别人的完整 skill/知识库，不要擅自删减
3. **论坛帖子也要完整** — 代码块、配置示例、注意事项一个都不能少
4. **支持文件夹结构** — 多文件素材直接建 `作者-主题/` 目录保留原始结构
5. **命名规则始终执行** — 即使原样保存，目录/文件名也要改为 `作者-主题` 格式
